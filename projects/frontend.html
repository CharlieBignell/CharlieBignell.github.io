Using React.js to Produce a Dashboard UI

March/April 2022

During this project, I continued the work on the stock market dashboard that's been the subject of the previous few projects. I used the web app from Project V, and implemented the UI I designed in Project IV, using React.js.

Unfortunately, due to circumstances beyond my control, I've had to put this project on hold. I have yet to complete the entire UI, but I've made good progress that I'm proud of, so I'll showcase here what I've done so far. I'll also talk about the remaining features, that I'll hopefully get around to completing sometime later this year.

Overview Page

The overview page is almost complete, the only missing element is the 'enter data' card in the top right. I left this until last as it's a little more complicated due to the fact that it needs to pass data to the backend and store it properly so it can be access in the future.

The current state of the overview page



A few notes about this page:

  • I took great care to ensure the page scales to different screen sizes. It isn't compatible with really small screens, e.g. mobile devices, but the cards, text etc. are dynamically sized.
  • I'm really happy with the header. It looks almost identical to the original design, and the 'time in the market' counter on the right ticks live from the datetime of the first transaction.
  • The time selector works well too, I used a third party component for this, that I'll credit at the bottom of this page.
  • The line graph works as intended - the time selector changes the x-axis, and the dropdown allows you to select different moving averages. Switching the toggle in the top right changes the lines to be the different markets e.g. SPY, NASDAQ etc., however the y-axis is fixed to Â£, whereas it should really be % for this view.
  • The treemap sections change colour based on the return in the selected time period. They're all green in the screenshot because I purposefully chose well-performing stocks and the view spans 10 years. This is what it looks like in a smaller scale:


  • Also on the treemap, if the box is too thin for the text to fit, the stock ticker is replaced with "..."
  • The summary card is a lot simpler than the original design. I realised the line graph background in the design was redundant, as it would just match the line graph on the right of it, so this was an intentional change. However, the card itself is quite boring, and I didn't get around to implementing a gauge for the volatility, which would be a good feature as the raw 'VIX' value isn't particularly helpful to see at first glance. Having said all this, all the necessary components are present
  • The pie chart has two parts, the inner chart represents sectors and the outer represents industries within those sectors.
  • As the pie chart gets more populated, having a legend (or visible labels as in the design) gets a bit cluttered. Therefore I made the intentional choice to remove all labels, and rely on tooltips to inspect what each section represents.
  • Speaking of tooltips, all the graphs on this page have tooltips to give more detailed information. The tooltips follow the mouse as it moves, here's a few examples of what they look like:



Breakdown and Details Page

Unfortunately, I only completed the area graph for the breakdown page. It shows the composition of the portfolio over time, and you can hover over to see a tooltip providing the individual values. The next step is to add the stock summary, and the 'winners and losers' cards. I also didn't get around to making the table on the details page.

Future Plans
